CodeSplash Homepage

Privacy Policy

Effective February 22, 2026

CodeSplash Homepage stores all data locally on your device. Nothing is collected, transmitted, or shared with anyone.

What This Extension Does

CodeSplash Homepage replaces your Chrome new tab page with a customizable homepage for organizing bookmarks. It is built for personal use and operates entirely within your browser.

Data Stored on Your Device

The following data is created and stored locally using your browser's IndexedDB and Chrome storage APIs:

  • Bookmarks (titles, URLs, display order)
  • Folders and folder organization
  • Custom icons uploaded by you
  • Background images uploaded by you
  • Layout and display settings

This data never leaves your device. It is not uploaded, synced, or backed up to any external server.

External Requests

Favicon fetching: When you add a bookmark, the extension requests a favicon image from Google's public favicon service (google.com/s2/favicons) using the bookmarked site's domain. No other data is sent in this request. This is the only network request the extension makes.

Permissions Used

The extension requests only the minimum permissions needed to function:

storage
Save bookmarks, folders, and settings locally
unlimitedStorage
Store custom icons and background images you upload
contextMenus
Add a right-click "Add to Homepage" option
activeTab
Read the current tab's URL and title when you add a bookmark

Changes to This Policy

If this policy is updated, the effective date at the top of this page will be revised. The extension does not collect contact information, so changes cannot be communicated directly. Please check this page periodically.

Contact
